A Most Disappointing Bug Report Button

I've been reading a lot of HackerNews the last few months. One of the ideas that I've pivked up from reading the comments their has been the idea of creating short feedback loops.

Traditionally changes to Parcel Details have been motivated by request from elected officals and internal complaints. This is a long feedback loop.

The question I'm struggling with is, "How do I shorten this feedback loop?"

My solution was to a large high constrast button to the footer of the website that was labeled "File a Bug Report". It contains a mailto link that directs you to the helpdesk email address.

When I implemented this bug report button. I limited it to just appearing to IP on our internal network because my position isn't rated to interact directly with the public.

I don't believe this is a significant issue, because more than have of the users for Parcel Details are browsing via IE 11, which is the County's standard browser. Implying that just targeting just internal users should be a large enough sample size to get good feedback.
